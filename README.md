# DevSkill Tracker

A Final Year Project that automatically measures developer skill growth from real coding activity.
DevSkill captures live coding sessions through a VS Code extension, classifies skill level with a
fine-tuned CodeBERT model, detects AI-assisted code, generates personalized learning quests, and
visualizes progress in a web dashboard.

**Live URLs**
- Frontend: https://fyp-ten-gray.vercel.app
- Backend API: https://hannan-12-devskill-backend.hf.space
- Model on Hugging Face Hub: https://huggingface.co/Hannan-12/devskill-codebert

## What it does

DevSkill turns raw coding activity into measurable, actionable skill data.

- **Real-time tracking** — A VS Code extension records keystrokes, file edits, time-on-task,
  language usage, and code snapshots while the developer works.
- **Skill classification** — A fine-tuned CodeBERT model labels each session (Beginner /
  Intermediate / Advanced) based on the code that was written. A scikit-learn fallback handles
  environments where the transformer isn't available.
- **AI-assistance detection** — Heuristic and statistical signals flag code that was likely
  generated by an LLM, so the skill score reflects what the developer actually wrote.
- **Personalized quests** — The backend uses the Anthropic Claude API to generate practice
  quests targeted at the user's weakest areas. Falls back to a curated quest pool if no key is
  configured.
- **Dashboards** — A React frontend visualizes session history, skill trends, completed quests,
  and per-language breakdowns for both students and admins.

## Architecture

```
+---------------------+        +-------------------------+        +-------------------+
|  VS Code Extension  | ─────► |   FastAPI Backend       | ◄────► |   Firestore DB    |
|  (TypeScript)       |  HTTP  |   (Python, HF Spaces)   |        |   (Google Cloud)  |
+---------------------+        +-----------+-------------+        +-------------------+
                                           │
                                           │ loads on startup
                                           ▼
                               +-------------------------+
                               |   CodeBERT model        |
                               |   (Hugging Face Hub)    |
                               +-------------------------+
                                           ▲
                                           │ HTTP
+---------------------+                    │
|   React Frontend    | ───────────────────┘
|   (Vite, Vercel)    |
+---------------------+
```

- The **VS Code extension** authenticates with Firebase, then streams session events to the
  FastAPI backend.
- The **backend** writes everything to **Firestore**, runs the CodeBERT classifier on submitted
  code, calls the **Anthropic API** when generating quests, and serves all data the frontend
  needs.
- The **frontend** reads from the backend over REST and authenticates against the same Firebase
  project.

## Local development

### Prerequisites
- Node.js 20+ and npm
- Python 3.13
- A Firebase project with Firestore enabled
- Service account JSON from Firebase Console saved to
  `backend/firebase_config/serviceAccountKey.json`

### 1. Backend
```bash
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python main.py
```
The API will be available at http://localhost:7860 (or `$PORT` if set).

### 2. Frontend
```bash
cd frontend
npm install
echo 'VITE_API_URL=http://localhost:7860' > .env.local
npm run dev
```
The dev server runs on http://localhost:5173.

### 3. Extension
```bash
cd extension
npm install
npm run compile
```
Then open the `extension/` folder in VS Code and press **F5** to launch an Extension
Development Host. Configure the backend URL in VS Code Settings → search "DevSkill" →
`devskill.api.baseUrl`.

## Environment variables

### Backend
| Name | Required | Description |
|---|---|---|
| `FIREBASE_CREDENTIALS_JSON` | yes (prod) | Full Firebase service account JSON. Falls back to `firebase_config/serviceAccountKey.json` if unset. |
| `FIREBASE_CREDS_PATH` | no | Override the local fallback path. |
| `ANTHROPIC_API_KEY` | no | Enables Claude-generated quests; gracefully falls back to a curated pool if missing. |
| `CORS_ORIGINS` | yes (prod) | Comma-separated list of allowed frontend URLs (e.g. `https://fyp-ten-gray.vercel.app`). |
| `CODEBERT_MODEL_ID` | no | Override the model loaded from HF Hub (default `Hannan-12/devskill-codebert`). |
| `PORT` | no | Port for uvicorn (default `7860`). |

### Frontend
| Name | Required | Description |
|---|---|---|
| `VITE_API_URL` | yes | Backend base URL. Set on Vercel to the HF Space URL. |

## Deployment

The project deploys to a fully **free** stack. Every component is connected to the GitHub repo
so updates flow automatically.

| Component | Host | Trigger |
|---|---|---|
| Frontend | Vercel (Hobby) | Auto-deploys on every push to `main` |
| Backend | Hugging Face Space (Docker, CPU basic free) | Run `./scripts/deploy-backend.sh` from local |
| Database | Cloud Firestore | Already managed |
| Model weights | Hugging Face Hub | Uploaded once, downloaded by backend on cold start |
| Extension | GitHub Releases | Built on `ext-v*` tags via Actions |

### One-time setup
1. **Firebase**: Create a project, enable Authentication and Firestore, download the service
   account JSON.
2. **Hugging Face Hub model**: Create a public model repo and upload the fine-tuned CodeBERT
   weights:
   ```bash
   huggingface-cli upload Hannan-12/devskill-codebert backend/ai_models/best_codebert_large/ . --repo-type=model
   ```
3. **Hugging Face Space**: Create a Docker SDK Space named `devskill-backend`. In its
   *Settings → Variables and secrets*, add:
   - Secret `FIREBASE_CREDENTIALS_JSON` — full JSON contents of the service account
   - Secret `ANTHROPIC_API_KEY` — optional
   - Variable `CORS_ORIGINS` — your Vercel URL
4. **Vercel**: Import the GitHub repo, set **Root Directory** to `frontend`, add env var
   `VITE_API_URL` pointing to the HF Space URL, deploy.

### First backend deploy
```bash
./scripts/deploy-backend.sh
```
This clones the HF Space, syncs `backend/` into it (excluding model weights, secrets, caches),
commits, and pushes. The Space then rebuilds its Docker image automatically.

## Updating a deployed component

| You change… | What to do |
|---|---|
| Frontend code | `git push` — Vercel auto-deploys |
| Backend code | `./scripts/deploy-backend.sh` |
| Model weights | Re-upload to HF Hub with `huggingface-cli upload …`. Backend picks them up on next cold start. |
| Extension code | Tag `ext-v0.0.x` and push the tag to trigger the build workflow, or run `vsce package` locally |
| Backend secret | Edit it in HF Space *Settings → Variables and secrets*; the Space auto-restarts |
| Allowed frontend origin | Update the `CORS_ORIGINS` variable on the HF Space |
